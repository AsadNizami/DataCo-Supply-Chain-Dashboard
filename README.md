## Project Overview

The raw dataset (`DataCoSupplyChainDataset.csv`) is 92 MB with 180,519 order rows. Because loading the full file in a browser would be impractical, a Python preprocessing script samples ~1,200 rows (~1.1 MB) and strips PII columns, producing a lightweight `data.json` that the React app loads at runtime.

Each chart has an **"AI Explain"** button. Clicking it captures a screenshot of the chart, sends it to Gemini 2.5 Flash (a vision-capable LLM), and opens a draggable chat panel with the analysis. You can then ask unlimited follow-up questions in the same panel — the model retains the full conversation context.

## Dataset & Preprocessing

### Source

The raw CSV (`archive/DataCoSupplyChainDataset.csv`) comes from the DataCo Smart Supply Chain dataset. It contains 180,519 rows and 53 columns covering orders, shipments, customers, products, and financials.

### Why preprocess?

- The full file is **92 MB** — too large to load in a browser efficiently.
- Several columns contain **PII** (customer email, name, password, address) that should not be served to a frontend.
- The target is a clean **~1–2 MB JSON** file holding a representative random sample.

### Running the preprocessor

A `preprocess.py` script is included at the project root. It reads the raw CSV, randomly samples 1,200 rows, keeps only the analytically relevant columns, and writes `data.json`.

```bash
# From the project root
python3 preprocess.py
```

This regenerates both `data.json` (project root) and `dashboard/public/data.json` (served by the dev server).

**Columns retained after preprocessing:**

| Column | Description |
|---|---|
| `Type` | Payment type (DEBIT, TRANSFER, etc.) |
| `Days for shipping (real)` | Actual days taken to ship |
| `Days for shipment (scheduled)` | Originally promised shipping days |
| `Benefit per order` | Gross benefit on the order |
| `Sales per customer` | Total sales attributed to that customer |
| `Delivery Status` | Advance shipping / On time / Late delivery / Canceled |
| `Late_delivery_risk` | Binary flag (1 = at risk) |
| `Category Name` | Product category |
| `Customer Country` | Country of the customer |
| `Customer Segment` | Consumer / Corporate / Home Office |
| `Department Name` | Selling department |
| `Market` | Geographic market (Europe, LATAM, Pacific Asia, etc.) |
| `Order City / Country / Region / State` | Order location info |
| `order date (DateOrders)` | Date the order was placed |
| `Order Item Discount Rate` | Discount applied (0–1) |
| `Order Item Profit Ratio` | Profit as a fraction of price |
| `Order Item Quantity` | Units ordered |
| `Sales` | Revenue on the line item |
| `Order Item Total` | Total value of the line item |
| `Order Profit Per Order` | Net profit for the order |
| `Order Status` | COMPLETE / PENDING / CANCELED / etc. |
| `Product Name` | Name of the product |
| `Product Price` | Listed product price |
| `Shipping Mode` | Standard Class / Second Class / First Class / Same Day |
| `shipping date (DateOrders)` | Actual ship date |

**Columns dropped (PII or redundant):**

`Customer Email`, `Customer Fname`, `Customer Lname`, `Customer Password`, `Customer Street`, `Customer Zipcode`, `Customer Id`, `Order Customer Id`, `Order Id`, `Order Item Id`, `Order Zipcode`, `Product Description`, `Product Image`, `Product Card Id`, `Latitude`, `Longitude`

## Installation

### Prerequisites

- **Node.js** ≥ 18 (tested on v24)
- **npm** ≥ 9
- **Python** ≥ 3.8 (for preprocessing only)
- A **Google Gemini API key** — free tier available at [aistudio.google.com](https://aistudio.google.com)

### Steps

```bash
# 1. Clone or download the project
cd DataCo-Supply-Chain-Dashboard

# 2. (Optional) Re-run preprocessing if you have the raw CSV
python3 preprocess.py

# 3. Install dashboard dependencies
cd dashboard
npm install

# 4. Configure your Gemini API key (see Configuration below)

# 5. Start the dev server
npm run dev
```

The app will be available at **http://localhost:5173** (or the next available port if 5173 is in use).

---

## Configuration

Copy the `.env` file in the `dashboard/` directory and fill in your key:

```bash
# dashboard/.env
VITE_GEMINI_API_KEY=your_gemini_api_key_here
```

Replace `your_gemini_api_key_here` with your actual key from [Google AI Studio](https://aistudio.google.com).

> **Important:** The `.env` file is intentionally excluded from version control. Never commit API keys. The `VITE_` prefix is required by Vite to expose the variable to the browser bundle.

After editing `.env`, restart the dev server for the change to take effect.

## Charts & Visualizations

The dashboard renders 9 charts plus 4 KPI summary cards.

### KPI Cards

| Metric | Description |
|---|---|
| Total Sales | Sum of all `Sales` values in the sample |
| Total Profit | Sum of all `Order Profit Per Order` values |
| Avg Order Value | Mean `Sales` per row |
| Late Delivery Rate | % of orders with `Late_delivery_risk = 1` or `Delivery Status = Late delivery` |

### Charts

| Chart | Type | Key insight |
|---|---|---|
| Sales by Category | Bar (vertical) | Which product categories drive the most revenue |
| Delivery Status Distribution | Donut pie | Share of on-time vs. late vs. canceled deliveries |
| Avg Profit by Shipping Mode | Bar | Profitability comparison across Standard / Second / First / Same Day |
| Total Sales by Market | Bar (horizontal) | Revenue breakdown across global markets |
| Late Delivery Rate by Region | Bar | Which regions have the highest delivery failure rates |
| Sales & Orders Over Time | Dual-axis line | Monthly revenue and order volume trends |
| Customer Segment Performance | Radar | Multi-metric comparison across Consumer / Corporate / Home Office |
| Order Status Breakdown | Pie | Distribution of COMPLETE / PENDING / CANCELED / etc. |
| Scheduled vs Actual Shipping Days | Scatter | Points above the diagonal line represent late deliveries |

## AI Assistant

Each chart has an **"AI Explain"** button in the bottom-right corner.

### How it works

1. Clicking the button captures a PNG screenshot of the chart using `html2canvas`.
2. The image is base64-encoded and sent to the **Gemini 2.5 Flash** model alongside a supply-chain analyst prompt.
3. A draggable floating chat panel opens with the AI's analysis.
4. You can type follow-up questions in the panel — the model retains the full conversation history (multi-turn chat via `model.startChat()`).

### Chat panel controls

| Control | Action |
|---|---|
| Drag the header bar | Move the panel anywhere on screen |
| `−` button | Minimize to title bar only |
| Refresh icon | Clear the current chat and start fresh |
| `×` button | Close the panel |
| Enter key | Send a follow-up message |

### Switching charts

Clicking "AI Explain" on a different chart starts a **new** chat session scoped to that chart. The previous session is replaced.

## Building for Production

```bash
cd dashboard
npm run build
```

The optimized output is written to `dashboard/dist/`. Serve it with any static file host (Nginx, Vercel, Netlify, etc.).

> **Note:** `data.json` in `public/` is copied into `dist/` automatically by Vite. The Gemini API key is baked into the bundle at build time via `import.meta.env.VITE_GEMINI_API_KEY` — ensure the `.env` file is present before running the build.
